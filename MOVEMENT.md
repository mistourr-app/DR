# Механика движения

## Последовательность хода

При клике игрока на клетку запускается следующая последовательность:

### 1. Проверка валидности хода
```javascript
// run.js: processPlayerMove()
if (!canMove) return;
```
- Проверка доступности клетки
- Проверка энергии для прыжков
- Проверка препятствий

### 2. Атака врагов "в спину"
```javascript
// run.js: processEnemyTurns()
const attackingEnemies = getEnemiesWhoCanAttack(player.pos.y);
for (const enemy of attackingEnemies) {
  dealDamageToPlayer(enemy.damage);
}
```
- Враги атакуют ДО начала движения
- Только враги с текущего ряда игрока
- Только враги с линией видимости
- Взаимный урон при ближнем бое

### 3. Проверка выживания
```javascript
if (player.hp <= 0) {
  // Игрок погиб, движение отменяется
  endRun('defeat');
  return;
}
```

### 4. Анимация движения
```javascript
// animation.js
startAnimation({
  type: 'move',
  duration: 200,
  onComplete: finalizeTurnAfterMove
});
```
- Плавное перемещение спрайта игрока
- Длительность 200ms
- Визуальная обратная связь

### 5. Обновление позиции
```javascript
// run.js: finalizeTurnAfterMove()
player.pos = targetPos;
```

### 6. Взаимодействие с новой клеткой
```javascript
// run.js: interactWithCell()
if (cell.type === 'enemy') {
  // Ближний бой
} else if (cell.type === 'heal') {
  // Подбор здоровья
} else if (cell.type === 'attack_bonus') {
  // Подбор бонуса
}
```

### 7. Завершение хода
```javascript
// run.js: finalizeTurnAfterMove()
turnOwner = 'player'; // Готов к следующему ходу
```

## Типы движения

### Бесплатное движение
- На прилегающие клетки (±1 по X, +1 по Y)
- Не тратит энергию
- Подсветка: зеленый цвет

### Платное движение (прыжок)
- На любую клетку следующего ряда (+1 по Y)
- Стоимость: 1 энергия за клетку расстояния
- Подсветка: голубой цвет

### Атака врага
- Клик на клетку с врагом
- Взаимный обмен уроном
- Подсветка: красный цвет

## Атака "в спину"

Враги атакуют игрока при уходе с их ряда:

### Условия атаки
1. Враг на том же ряду, что и игрок (до движения)
2. Враг видит игрока (нет стен между ними)
3. Враг жив (currentHp > 0)

### Механика
- Атака происходит ДО анимации движения
- Урон применяется мгновенно
- Если игрок погибает - движение отменяется
- Если игрок выживает - движение продолжается

### Визуальная обратная связь
- Красное мигание экрана
- Всплывающие цифры урона
- Анимация атаки врага

## Ближний бой

При клике на клетку с врагом **на следующем ряду** (ряд `player.pos.y + 1`):

### Условия
- Враг находится на ряду непосредственно перед игроком
- Клик на клетку с врагом (не на своего ряду!)

### Важное ограничение
- На врагов **на своём ряду** кликнуть для боя нельзя — можно только стрелять из арбалета или двигаться дальше
- На врагов **на ряду впереди** — ближний бой с перемещением

### Последовательность
1. Враги с текущего ряда атакуют "в спину" (до движения)
2. Проверка выживания игрока
3. Анимация перемещения игрока на клетку врага (250ms)
4. Короткий тычок (100ms) → взаимный обмен уроном (`processMeleeCombat`)
5. Игрок возвращается на позицию клетки
6. Если враг мёртв — анимация смерти (fade out + сдвиг вверх, 700ms)
7. Если игрок мёртв — экран поражения
8. Завершение хода

### Расчет урона
```javascript
// Урон игрока: текущее HP игрока + бонусы атаки
const playerDamage = player.hp + sum(attackBonuses);

// Урон врага: его текущий HP (враг бьёт в ответ, если жив)
const enemyDamage = enemy.currentHp;
```

### Смерть врага
- Анимация смерти (fade out)
- Удаление из карт угроз
- Флаг `threatMapsDirty = true`
- Очистка клетки

## Дальний бой (арбалет)

### Условия
- Враг в пределах дальности (≤3 клетки)
- Есть заряды арбалета
- Линия видимости не заблокирована

### Механика
- Урон: 3 + бонусы атаки
- Расход: 1 заряд
- Враг не отвечает
- Можно стрелять не двигаясь

### Последовательность
1. Проверка условий
2. Анимация выстрела
3. Нанесение урона
4. Расход заряда
5. Проверка смерти врага
6. Обновление карт угроз

## Карты угроз (Threat Maps)

Система отслеживания зон видимости врагов:

### Структура
```javascript
{
  alert: Map<cellKey, Set<enemyId>>,  // Враг видит игрока
  idle: Map<cellKey, Set<enemyId>>    // Враг в зоне видимости
}
```

### Обновление
- При смерти врага: удаление из всех клеток
- При движении игрока: пересчет alert/idle
- Флаг `threatMapsDirty` для оптимизации

### Визуализация
- Alert: оранжевый 15%
- Idle: красный 15%
- Наложение от нескольких врагов

## Боссфайт

Особая механика для последних 2 рядов:

### Отличия от обычного движения
- Горизонтальное перемещение (только по X)
- Пошаговая система (игрок → босс)
- Клетки атаки (5-10 урона)
- Динамическая генерация клеток

### Последовательность хода
1. Ход игрока (движение или атака)
2. Проверка победы над боссом
3. Ход босса (AI)
4. Проверка поражения игрока
5. Генерация новых клеток

### AI босса
Три профиля поведения:
- **Агрессивный** (>60% HP): атака и сближение
- **Сбалансированный** (25-60% HP): атака и защита
- **Защитный** (<25% HP): защита и отступление

## Энергия

### Восстановление
- Не восстанавливается автоматически
- Только через предметы (+10 энергии)

### Расход
- Прыжок: 1 энергия за клетку расстояния
- Пример: прыжок на 2 клетки = 2 энергии

### Стратегия
- Экономия для критических прыжков
- Подбор предметов энергии
- Планирование маршрута

## Анимации

### Движение игрока
- Длительность: 200ms
- Плавная интерполяция
- Callback после завершения

### Атака врага
- Длительность: 300ms
- Визуальный эффект
- Красное мигание экрана

### Смерть врага
- Длительность: 400ms
- Fade out эффект
- Удаление после завершения

### Выстрел из арбалета
- Длительность: 250ms
- Визуальная траектория
- Эффект попадания

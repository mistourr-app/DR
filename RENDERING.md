# Архитектура рендеринга

## Слоистая система отрисовки

Рендеринг игрового поля построен на принципе разделения на независимые слои, каждый из которых отвечает за свою визуальную функцию.

### Слои (снизу вверх)

#### 1. Фон клеток (Cell Background)
- **Функция**: Статичная сетка игрового поля
- **Цвет**: `#1a1d28` (темно-серый)
- **Границы**: `#2d313d` (светло-серый)
- **Особенности**: 
  - Единый цвет для всех клеток
  - Рисуется на статичных координатах (cellY)
  - Не зависит от анимаций

#### 2. Подсветка угроз (Threat Highlights)
- **Функция**: Визуализация зон видимости врагов
- **Цвета**:
  - Оранжевый (`rgba(255, 165, 0, 0.15)`) - враг в режиме alert
  - Красный (`rgba(255, 0, 0, 0.15)`) - враг в режиме idle
- **Особенности**:
  - Прозрачность 15% для наложения
  - Несколько врагов → суммирование прозрачности
  - Динамическое обновление при смерти врагов
  - Рисуется на статичных координатах (cellY)

#### 3. Объекты (Cell Content)
- **Функция**: Игровые объекты (враги, предметы, стены)
- **Стиль**: Карточки без фона
- **Границы**: Только для врагов, стен, босса
- **Особенности**:
  - Рисуется на динамических координатах (drawY)
  - Поддержка анимаций
  - Прозрачность для пройденных рядов

#### 4. Игрок (Player)
- **Функция**: Карточка игрока
- **Стиль**: С фоном и границей
- **Особенности**:
  - Рисуется на динамических координатах (drawY)
  - Поддержка анимаций движения

### Система координат

#### Статичные координаты (cellY)
```javascript
const cellY = getRowY(y);
```
- Используются для фона и подсветок
- Не зависят от анимаций
- Фиксированная позиция на сетке

#### Динамические координаты (drawY)
```javascript
const drawY = cellY + (cell.animOffsetY || 0);
```
- Используются для объектов и игрока
- Учитывают анимации движения
- Плавные переходы между клетками

### Карты угроз (Threat Maps)

Система отслеживания зон видимости врагов:

```javascript
{
  alert: Map<cellKey, Set<enemyId>>,
  idle: Map<cellKey, Set<enemyId>>
}
```

**Ключевые особенности:**
- Каждая клетка хранит Set врагов, которые её видят
- При смерти врага его ID удаляется из всех клеток
- Мертвые враги (currentHp <= 0) исключаются из расчетов
- Флаг `threatMapsDirty` для оптимизации пересчета

### Функции рендеринга

#### drawCellBackground(ctx, x, y, cellSize)
Рисует статичный фон клетки:
- Заливка `#1a1d28`
- Граница `#2d313d` толщиной 1px

#### drawThreatHighlight(ctx, x, cellY, cellSize, threatMaps)
Рисует подсветку угроз:
- Проверяет наличие врагов в alert/idle картах
- Накладывает цвета с прозрачностью 15%
- Позволяет суммирование от нескольких врагов

#### drawCellContent(ctx, cell, x, drawY, cellSize, isPlayerRow)
Рисует содержимое клетки:
- Без фона (прозрачно)
- Границы только для врагов, стен, босса
- Эмодзи и текст контента
- Прозрачность для пройденных рядов

### Последовательность отрисовки

```javascript
// 1. Фон всех клеток
for (let y = startRow; y <= endRow; y++) {
  for (let x = 0; x < 5; x++) {
    drawCellBackground(ctx, x, y, cellSize);
  }
}

// 2. Подсветка угроз
for (let y = startRow; y <= endRow; y++) {
  for (let x = 0; x < 5; x++) {
    drawThreatHighlight(ctx, x, cellY, cellSize, threatMaps);
  }
}

// 3. Объекты
for (let y = startRow; y <= endRow; y++) {
  for (let x = 0; x < 5; x++) {
    if (!isPlayer) {
      drawCellContent(ctx, cell, x, drawY, cellSize, isPlayerRow);
    }
  }
}

// 4. Игрок (поверх всего)
drawCellContent(ctx, playerCell, px, playerDrawY, cellSize, true);
```

### Оптимизация

- **Оконный рендер**: Отрисовка только видимых рядов (startRow → endRow)
- **Dirty flags**: Пересчет карт угроз только при изменениях
- **Статичные слои**: Фон и подсветки не пересчитываются при анимациях объектов

### Визуальная иерархия

1. **Фон** - нейтральная основа (#1a1d28)
2. **Угрозы** - тактическая информация (оранжевый/красный 15%)
3. **Объекты** - игровые элементы (карточки без фона)
4. **Игрок** - фокус внимания (карточка с фоном)

Эта архитектура обеспечивает:
- Четкую визуальную иерархию
- Читаемость тактической информации
- Плавные анимации без артефактов
- Корректное наложение эффектов

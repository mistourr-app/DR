# Dungeon Card Crawler — Контекст проекта

## Обзор проекта

**Dungeon Card Crawler** — это мобильный пошаговый fast-paced dungeon crawler в жанре rogue-lite, реализованный как PWA-приложение на чистом JavaScript с HTML5 Canvas.

### Целевая аудитория
Мужчины 18+, которые хотят получить опыт быстрого dungeon crawler на мобильных устройствах.

### Уникальная особенность
Очень быстрый геймплей, близкий к раннерам, но с тактической глубиной.

### Технологии
- Чистый JavaScript (ES6 модули)
- HTML5 Canvas для рендеринга
- Tailwind CSS для UI
- PWA (Progressive Web App) с manifest.json и service worker
- localStorage для сохранения мета-прогресса

## Архитектура

### Модульная структура

| Модуль | Описание |
|--------|----------|
| `main.js` | Точка входа, игровой цикл (game loop), обработка кликов, resize |
| `config.js` | Константы: DIMS (размеры), AppState (состояния приложения) |
| `state.js` | Глобальное состояние: appState, runState, metaState; сохранение/загрузка меты |
| `run.js` | Логика забега: движение, взаимодействие с клетками, последовательность хода |
| `combat.js` | Боевая система: расчёт урона, бонусы атаки/защиты, ближний/дальний бой |
| `enemyAI.js` | Логика врагов: проверка видимости, атаки, карты угроз (threat maps) |
| `bossAI.js` | AI босса с тремя профилями поведения (агрессивный/сбалансированный/защитный) |
| `renderer.js` | Слоистая отрисовка: фон → угрозы → объекты → игрок → всплывающий текст |
| `ui.js` | UI экраны: меню уровней, победа/поражение, верхняя/нижняя панели |
| `animation.js` | Система анимаций (плавное перемещение, fade out, и т.д.) |
| `events.js` | Система событий (RUN_STARTED, PHASE_CHANGED, и т.д.) |
| `registry.js` | Регистры: уровни, враги, клетки, объекты |
| `utils.js` | Утилиты: PRNG (детерминированная генерация), генерация объектов арены |
| `tutorial.js` | Система обучения (12 шагов для tutorial уровня) |
| `log.js` | Логирование |
| `admin.html` | Админ-панель (скрытый вход: 5 кликов по заголовку в меню) |

### Состояния приложения (State Machine)

```
BOOT → META_HUB → RUN_PLAYING → RUN_SUMMARY / RUN_VICTORY → META_HUB
```

### Фазы уровня
- **dungeon** — вертикальный забег по рядам вверх (5 колонок)
- **boss_arena** — бой с боссом (последние 2 ряда, горизонтальное перемещение)

## Игровые механики

### Игрок
- **HP**: 20 (максимальное)
- **Энергия**: 10 (максимальная)
- **Оружие**: арбалет (урон 3, дальность 3, заряды 3)
- **Инвентарь**: 2 слота бонусов атаки + 2 слота бонусов защиты

### Движение
| Тип | Описание | Стоимость |
|-----|----------|-----------|
| Бесплатное | На прилегающие клетки (±1 X, +1 Y) | 0 энергии |
| Прыжок | На любую клетку следующего ряда (+1 Y) | 1 энергия за клетку расстояния по X |
| Ближний бой | Клик на врага на следующем ряду (`player.pos.y + 1`) | 0 энергии, враги текущего ряда атакуют «в спину» перед боем |
| Дальний бой | Клик на врага на своём ряду (в пределах дальности арбалета) | 1 заряд арбалета, враг не отвечает |

### Последовательность хода
1. Атака врагов "в спину" (враги с текущего ряда атакуют ДО движения)
2. Проверка выживания игрока
3. Анимация движения (250ms) или боя
4. Обновление позиции
5. Взаимодействие с новой клеткой (подбор предметов, бой)
6. Завершение хода

### Анимация атаки врагов "в спину"
- Враг делает выпад по оси X в сторону игрока (30% клетки, 150ms)
- Возврат на место (150ms)
- Несколько врагов атакуют по очереди

### Анимация ближнего боя
- Игрок перемещается на клетку врага (250ms)
- Короткий тычок (100ms) → обмен уроном
- Игрок возвращается на позицию клетки (100ms)
- Если враг мёртв — fade out + сдвиг вверх (700ms)

### Враги
| Тип | HP | Дальность обзора | Дальность атаки |
|-----|----|-----------------|-----------------|
| Снайпер (TYPE_1) | 4 | 4 | 4 |
| Страж (TYPE_2) | 8 | 2 | 2 |

### Объекты на карте
- **Стена** — блокирует движение и видимость
- **Здоровье** — восстановление +6 HP
- **Энергия** — восстановление +10 энергии
- **Заряды** — +2 патрона для арбалета
- **Золото** — мета-валюта (+5)
- **Бонус атаки** — +5 к урону (2 слота)
- **Бонус защиты** — +5 к защите (2 слота)
- **Клетка атаки** — 5-10 урона (только в боссфайте)

### Боссфайт
- Горизонтальное перемещение (только по X)
- Пошаговая система: игрок → босс
- Три способа атаки: клетки атаки, арбалет, ближний бой бонусами
- AI босса меняет поведение в зависимости от HP:
  - **Агрессивный** (>60% HP): атака и сближение
  - **Сбалансированный** (25-60% HP): атака и защита
  - **Защитный** (<25% HP): защита и отступление

### Карты угроз (Threat Maps)
- **Alert** — враг видит игрока (оранжевая подсветка 15%)
- **Idle** — враг в зоне видимости (красная подсветка 15%)
- Наложение от нескольких врагов (суммирование прозрачности)

## Обучение (Tutorial)

12 интерактивных шагов:
1. Движение вперёд
2. Движение в сторону за энергию
3. Подбор здоровья
4. Подбор патронов
5. Ближний бой
6. Дальний бой из арбалета
7. Атака в спину
8. Бонус атаки
9. Бой с бонусом
10. Арена босса
11. Клетка атаки
12. Победа над боссом

Особенности: нельзя ошибиться, автоматический прогресс, подсказки на canvas.

## Соглашения и паттерны

### Код
- ES6 модули (`import`/`export`)
- Центральное состояние через `gameState` в `state.js`
- Событийная система через `events.js`
- Анимации через `animation.js` (плейсы с callbacks)

### Рендеринг
- Адаптивный размер canvas под экран
- Масштабирование элементов через `fontScale = DIMS.CELL_SIZE / 64`
- Четырёхпроходный рендеринг: фон → угрозы → содержимое → игрок → текст

### Naming conventions
- Константы: UPPER_SNAKE_CASE (`OBJECT_TYPES`, `CELL_DEFS`)
- Функции: camelCase (`processPlayerAction`, `dealDamageToBoss`)
- Состояния: camelCase (`runState`, `metaState`)
